WHAT IS THIS?
-------------
A mobile-first restaurant ordering system where customers scan QR codes, 
browse menu, and place orders from their phones.

Backend: Google Apps Script (Serverless)

KEY FEATURES
------------
For Customers:
- Scan QR code on table
- Browse menu with search
- Add items to cart
- Place order with details
- Email confirmation
- Track order status
- Add more items to order

For Owner:
- Admin dashboard
- View today's orders
- Manage menu (CRUD)
- Statistics display
- Email notifications
- Role-based access

Technical:
- Mobile-only (5-factor detection)
- Auto sheet rotation (10K orders)
- Menu caching (5 minutes)
- Email automation
- Real-time tracking

HOW IT WORKS
------------
1. Customer scans QR code
2. Website loads menu from Google Sheets
3. Customer adds items to cart
4. Places order with details
5. Backend saves to Google Sheets
6. Emails sent to customer & owner
7. Order can be tracked by mobile/ID

DATABASE (Google Sheets)
------------------------
Sheet 1: menu1
- Columns: ID, Name, Description, Price, Category, Image, Available

Sheet 2: orders
- Columns: Timestamp, Order ID, Table, Customer, Mobile, Email, Items, Total
- Auto-rotation at 10,000 rows

Sheet 3: EmailQueue (Optional)
- For email queuing and retry
